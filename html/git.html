Git Branching Strategy

Master:

Represents the production-ready code. Only updated with code from release branches or hotfixes.

Develop:

Integration branch where ongoing development and feature work occurs. Feature branches are merged into this branch when features are complete.

Feature Branches:

Created off the develop branch for developing new features. Merged back into the develop branch when the feature is complete.

Release Branches:

Created from the develop branch when preparing for a new release. Used for final testing and bug fixing before merging into master. Merged into both master and develop when the release is ready.

Hotfix Branches:

Created from the master branch to quickly address critical issues in production. Merged into both master and develop to ensure fixes are included in future releases.

Reverting the Latest Commit:

latest commit revert below command

git revert HEAD

Reverting a Specific Previous Commit:

git log
git revert commit-hash

What is Rebase

In Git, rebasing is a process that allows you to modify the commit history by moving, combining, or altering the order of commits. The primary goal of rebasing is to create a linear commit history, making it easier to understand and navigate the project's development timeline
