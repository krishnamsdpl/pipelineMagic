Git Branching Strategy

Master:

Represents the production-ready code. Only updated with code from release branches or hotfixes.

Develop:

Integration branch where ongoing development and feature work occurs. Feature branches are merged into this branch when features are complete.

Feature Branches:

Created off the develop branch for developing new features. Merged back into the develop branch when the feature is complete.

Release Branches:

Created from the develop branch when preparing for a new release. Used for final testing and bug fixing before merging into master. Merged into both master and develop when the release is ready.

Hotfix Branches:

Created from the master branch to quickly address critical issues in production. Merged into both master and develop to ensure fixes are included in future releases.

Reverting the Latest Commit:

latest commit revert below command

git revert HEAD

Reverting a Specific Previous Commit:

git log
git revert commit-hash

What is Rebase

In Git, rebasing is a process that allows you to modify the commit history by moving, combining, or altering the order of commits. The primary goal of rebasing is to create a linear commit history, making it easier to understand and navigate the project's development timeline

Understanding Git Merge, Rebase, and Squash

1. Git Merge

Merge is a Git command used to combine changes from one branch into another. It creates a new commit that has two parent commits, preserving the commit history of both branches. While it's straightforward, it can result in a more complex and "noisy" commit history. 

            
    git checkout main
    git merge feature-branch

2. Git Rebase

Rebase is another integration strategy that involves rewriting commit history by applying the changes of one branch on top of another. It produces a linear and cleaner commit history, reducing "merge commits" and making the history easier to follow. However, rewriting history can be risky, especially if changes have been shared with others. 

            
    git checkout feature-branch
    git rebase main
    git checkout main
    git merge feature-branch

3. Git Squash

Squash is used to combine multiple commits into a single commit before integrating changes into another branch. It collapses multiple commits into one, creating a cleaner and more concise commit history. However, it comes with the trade-off of losing the individual commit history, which may be important for detailed tracking. 

            
    git checkout main
    git merge --squash feature-branch

It's important to choose the integration strategy that aligns with your project's workflow and collaboration practices. Communicate and agree with your team on the preferred strategy to maintain a clear and manageable commit history in your Git repository.
