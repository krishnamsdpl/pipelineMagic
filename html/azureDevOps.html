Azure DevOps

Azure DevOps Concepts

  1. Azure DevOps Services:
    1. Azure Repos: A version control system that supports Git repositories and Team Foundation Version Control (TFVC). It allows teams to manage and track code changes efficiently.
    2. Azure Pipelines: A continuous integration/continuous deployment (CI/CD) service that automates the build, test, and deployment phases of the application development lifecycle.
    3. Azure Boards: An agile project management and work tracking system. It supports Scrum, Kanban, and custom processes, allowing teams to plan, track, and discuss work across the entire development process.
    4. Azure Test Plans: A testing tool that allows teams to plan, track, and manage testing efforts. It supports manual and automated testing.
    5. Azure Artifacts: A package management system that enables teams to create, share, and consume packages (such as NuGet or npm packages) from public and private sources.
  2. Azure DevOps Server: Formerly known as Team Foundation Server (TFS), Azure DevOps Server is the on-premises version of Azure DevOps. It provides similar capabilities as Azure DevOps Services but can be hosted on an organization's own infrastructure.
  3. Azure DevOps Organizations and Projects: An organization in Azure DevOps represents a collection of projects, users, and settings. Within an organization, projects are used to group related work items, repositories, pipelines, and other resources.
  4. Repositories: Azure DevOps Repositories host source code and version history. You can choose between Git repositories (distributed version control) or Team Foundation Version Control (TFVC) repositories (centralized version control).
  5. Pipelines: Azure Pipelines enable the creation of automated build and release pipelines. These pipelines can be defined using YAML or through a visual designer, allowing teams to automate the process of building, testing, and deploying applications.
  6. Build Agents: Build agents are used to run build and release tasks defined in pipelines. Agents can be hosted in the cloud or on-premises and are responsible for executing the build and deployment processes.
  7. Build Artifacts: Build artifacts are the output of the build process, such as binaries, packages, or deployment scripts. These artifacts can be published and consumed by other pipelines.
  8. Service Connections: Service connections in Azure DevOps enable integration with external services such as Azure, GitHub, Docker Hub, etc. They allow pipelines to securely access and interact with these external services.
  9. Environment and Deployment: Environments in Azure Pipelines represent logical deployment targets. Pipelines define the stages and conditions for deploying applications to different environments.
  10. Release Gates: Release gates allow teams to include automated checks or approvals in the deployment pipeline before progressing to the next stage. Gates help ensure the quality and readiness of the application before moving to production.
  11. Azure Boards Work Items: Work items in Azure Boards represent tasks, bugs, features, or other units of work that teams plan, track, and manage throughout the development process.
